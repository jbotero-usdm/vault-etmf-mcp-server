Datasources

The current environment uses three custom datasources for documents, objects, and security readiness.

Documents

vaultetmfv2documents
  • TMF document content and metadata indexing
  • Supports retrieval, filing context, and indexing health
  • Feeds document-level search and filing workflows

Objects

vaultetmfv2objects
  • Study, study country, and site object records
  • Supports filing context and operational visibility
  • Indexed as documents in this environment

Security

vaultetmfv2security
  • User, group, and membership readiness snapshot
  • Operational and permissions context
  • Not presented as a business-facing searchable corpus

Human oversight model

AI acts as a delegate to the user, not as an autonomous final decision-maker. Where outputs affect regulated workflows, a qualified human remains responsible for review, acceptance, rejection, or escalation.

Agent Usage

The ABC BioPharma TMF Filer Agent is currently configured as a gather-only filing assistant.

Current behavior

  • Identify the relevant Box file
  • Resolve Vault classification
  • Resolve study, country, and site identifiers
  • Prepare filing inputs for downstream review

Current boundary

The current agent does not perform the final autonomous filing step. It remains intentionally separate from admin crawl, reindex, and troubleshooting operations.

Admin Ops

Administrator-only maintenance and diagnostics for indexing and environment operations.

Admin-only actions

  • Full Crawl
  • Reindex
  • Smoke Test
  • Detailed Troubleshooting
  • Run History and Diagnostics

These capabilities should be restricted to designated environment administrators.
